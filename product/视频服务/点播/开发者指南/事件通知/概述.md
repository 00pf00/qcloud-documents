对云点播中的视频发起的上传、删除、视频处理等操作，都可以被称为一个事件。事件的执行需要一段时间才能完成。云点播在事件结束时，会立即通知 App 服务操作的执行结果，即事件通知。

事件通知有以下两种方式：

* 普通回调：在控制台上配置一个回调 URL，点播平台在事件完成后向 URL 发送 HTTP 请求，请求体中包含通知内容，具体请参见 [普通回调](【超链接到本文的“普通回调”一章】) 单元；
* 可靠回调：点播平台在事件完成后，将通知内容放入点播系统内置的的队列，App 服务通过服务端 API 消费队列中的通知，具体请参见 [可靠回调](【超链接到本文的“可靠回调”一章】) 单元。

通过控制台的 [回调配置](https://cloud.tencent.com/document/product/266/14058#.E5.9B.9E.E8.B0.83.E9.85.8D.E7.BD.AE) 页面，可以选择【回调模式】是“普通回调”还是“可靠回调”，并在【事件回调配置】中选择启用哪些事件通知。

## 普通回调

普通回调，是 App 服务被动接收事件通知的模式。配置回调 URL 并选择普通回调模式后，云点播会在事件完成后，向回调 URL 发起回调。

云点播发起的普通回调的形式是 HTTP 请求，请求体为 JSON 格式，内容为不含 EventHandle 参数的 [EventContent](/document/api/266/31773#EventContent) 结构。

以 [任务状态变更通知]() 为例，回调中的 EventType 参数为 ProcedureStateChanged，信息由 ProcedureStateChangeEvent 参数表示（[ProcedureTask](https://cloud.tencent.com/document/api/266/31773#ProcedureTask) 结构）。

## 可靠回调
可靠回调，是 App 服务主动向云点播拉取事件通知的模式。选择可靠回调模式后，云点播将把事件通知放入队列中。App 服务通过服务端 API 从队列中依次消费事件通知。

* [拉取事件通知](/document/product/266/33433)
* [确认事件通知](/document/product/266/33434)

App 服务通过“拉取事件通知” API 获取消息后，需要调用“确认事件通知” API 进行确认。因为消息必须被确认之后，才会从云点播中的队列中删除，所以“可靠回调”的可靠性高于“普通回调”。如果对事件通知的可靠性要求高，建议使用“可靠回调”模式。
