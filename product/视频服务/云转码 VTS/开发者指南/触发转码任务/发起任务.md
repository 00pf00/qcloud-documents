在VTS中，有两种发起转码任务的方式：一种是通过设置工作流，实现文件上传后自动触发任务；另一种是通过调用API发起单个文件转码任务。第一种方式请参考[工作流]()进行设置。本文将介绍后一种任务发起方式。

### 发起单个任务
开发者可以通过API [ProcessMedia](/document/product/1090/33672)来发起单个文件转码任务，具体参数含义请参考该API接口文档。如果调用成功，该接口返回为该任务的ID，即结果中的TaskId字段。

请求示例：
```
https://vts.tencentcloudapi.com/?Action=ProcessMedia
&InputInfo.Type=COS
&InputInfo.CosInputInfo.Bucket=TopRankVideo-125xxx88
&InputInfo.CosInputInfo.Region=ap-chongqing
&InputInfo.CosInputInfo.Object=/movie/201907/WildAnimal.mov
&MediaProcessTask.TranscodeTaskSet.0.Definition=20
&MediaProcessTask.TranscodeTaskSet.1.Definition=30
&MediaProcessTask.TranscodeTaskSet.2.Definition=40
&<公共请求参数>
```

返回结果示例：
```
{
  "Response": {
    "RequestId": "6ca31e3a-6b8e-4b4e-9256-fdc700064ef3",
    "TaskId": "125xxx65-procedurev2-bffb15f07530b57bc1aabb01fac74bca"
  }
}
```

如果开发者设置了事件通知CMQ，则将会接收到该任务的完成事件通知。除了通过CMQ接收事件通知消息，也可以通过[查询任务详情]() API 接口查询任务结果，输入参数即为ProcessMedia接口返回的TaskId。
