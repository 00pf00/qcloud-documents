一个完整的腾讯云 API 请求需要两类请求参数：公共请求参数和接口请求参数。不同请求接口需要用到的请求参数不同，详情请参见 [API 概览](https://cloud.tencent.com/document/product/655/32373)。下面将为您介绍腾讯云 API 请求需要用到的6个公共请求参数。

## 公共请求参数
公共请求参数是每个接口都需要用到的请求参数，开发者每次使用腾讯云 API 发送请求时，都需要携带这些公共请求参数，否则会导致请求失败。公共请求参数的首字母均为大写，以此区分于接口请求参数。

## 参数列表
>! 文章中的接口实例以身份安全验证为例，腾讯云各产品的使用方式，请参考各产品对应的文档。

参数名称 | 描述	| 类型	| 必选
- | - | - | -
Action	| 具体操作的指令接口名称<br>例如身份安全验证中， 用户调用身份安全验证接口，则 Action 参数即为 ap-guangzhou	| String	| 是
Region	| 地域参数，用来标识希望操作哪个地域的实例，默认填为all	| String	| 否
Timestamp	| 当前 UNIX 时间戳，可记录发起 API 请求的时间	| UInt	| 是
Nonce	| 随机正整数，与 Timestamp 联合起来， 用于防止重放攻击。	| UInt	| 是
SecretId	| 在 [云API密钥](https://console.cloud.tencent.com/cam/capi) 上申请的标识身份的 SecretId，一个 SecretId 对应唯一的 SecretKey , 而 SecretKey 会用来生成请求签名 Signature。详情请参见 [签名方法](https://cloud.tencent.com/document/product/215/1693) 章节	| String	| 是
Signature	| 请求签名，用来验证此次请求的合法性，需要用户根据实际的输入参数计算得出。计算方法详情请参见 章节	| String	| 是

## 使用示例
腾讯各云产品 API 请求链接中，公共请求参数的形式如下，以身份安全验证为例，其请求链接的形式为：
```
https://csec.api.qcloud.com/v2/index.php?Nonce=864243805
&name=%E4%BB%BB%E5%AE%AA%E4%BC%9F
&Timestamp=1545738989
&Region=all
&idNumber=231083198101033217
&SecretId=AKIDmQtAxYTAB2iBS8s2DCzazCD2g7OUq4Zw
&Signature=i/en5pXSdRTKUACqFEeX805xgnY%3D
&Action=BspIdCardAuth
&orderNo=12312312
```
