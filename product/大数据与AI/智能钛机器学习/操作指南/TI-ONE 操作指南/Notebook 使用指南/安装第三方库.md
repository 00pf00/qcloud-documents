## 操作场景
本文档将向您演示如何在 Notebook 中**查看已有依赖包**以及**安装第三方库**。

## 操作步骤
### 查看已有依赖包
通过在Terminal文件中输入**pip list**命令行，查看镜像内已有的依赖包；
![](https://main.qcloudimg.com/raw/1394023d944f66f15233b076058b0543.png)



### 安装外部依赖包

#### A. 依赖包在镜像中已有

如果期望的安装包已在内置的依赖包列表中，您可以使用**pip install --user**命令来安装内置列表的依赖包。 后续您可直接输入import命令使用。

> 注意：当用户创建Notebook时选择开启了root权限，并在Notebook使用“sudo su -”指令切换到root了，那么pip install命令将不需要添加 --user。）

![](https://main.qcloudimg.com/raw/e0dffd92ba1f9016dd25ecb03c2aac66.png)   

#### B. 依赖包在镜像中并不存在

此时如果直接使用**pip install**命令将会报错。您需要：首先，将所需的第三方依赖包从外部网络下载。其次，上传依赖包。最后，新建terminal文件并通过pip install 命令安装上传的外部包。上述步骤成功后，后续您才能通过import命令使用。 ![](https://main.qcloudimg.com/raw/d3b3fe88ed67815a19419f251a3c5cd8.png)  

(1) 点击“上传文件”按钮，将依赖包上传。上传大小限制300MB。

 ![](https://main.qcloudimg.com/raw/7371e81145b0a47544d6d087cf75efb3.png)



(2) 新建terminal文件，并输入“pip install <包名> --user”命令安装。 

![](https://main.qcloudimg.com/raw/f855b502429c8bfd473cd78b4f3b3402.png)

![](https://main.qcloudimg.com/raw/b30a16e0acb88ab1ad049e4ce3dea51a.png)

###  注意事项

A. 如果系统出现以下提示代表需要升级pip，您需要先通过在terminal文件中输入“pip install --upgrade pip --user”来进行升级，升级后再进行 “pip install”操作。 

![](https://main.qcloudimg.com/raw/8076fe29324e3fb55aa0d812982d401b.png)

B. 安装完依赖包后，您需要重启kernel才能进行import操作。   ![](https://main.qcloudimg.com/raw/3e72d7e56d4fcf96f44b4ea6163f1c19.png)

C. 如果您创建Notebook时选择开启了root权限，并在Notebook使用“sudo su -”指令切换到root，那么pip install命令将不需要添加 --user。
