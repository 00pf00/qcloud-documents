
CDWCH 节点按功能不同，分为计算节点和Zookeeper节点（以下简称“ZK节点”）。
- 高可用模式下， 数据节点个数须为偶数，ZK节点个数为3。
- 非高可用模式下，集群不配置ZK节点，只有计算节点。
 - 对于计算节点的选择建议如下：CPU是关键指标，主频越⾼越好，通常建议配置32核以上的机型 。内存越⼤，PageCache加速作⽤越明显，建议配置128GB以上内存。 
 - 对于ZK节点，其计算和存储的负载与集群的规模成正⽐。当数据规模在TB级别时，建议ZK节点选择4核以上机型+SSD盘。
数据节点或ZK节点的规格类型如下表：
 

| 类型    | CPU  | 内存  | 系统盘 |
| ------- | ---- | ----- | ------ |
| 标准型1 | 2核  | 4GB   | 50GB   |
| 标准型2 | 4核  | 16GB  | 50GB   |
| 标准型3 | 8核  | 32GB  | 50GB   |
| 标准型4 | 16核 | 64GB  | 50GB   |
| 标准型5 | 24核 | 96GB  | 50GB   |
| 标准型6 | 32核 | 128GB | 50GB   |
| 标准型7 | 64核 | 256GB | 50GB   |

存储规格和其适用场景如下：

 

| 存储      | 容量          | 适用场景                                                     |
| --------- | ------------- | ------------------------------------------------------------ |
| 高效云盘  | 100GB~32000GB | 高性能云硬盘适用于高数据可靠性要求、普通中度性能要求的 Web/App 服务器、业务逻辑处理、中小型建站等中小型应用场景。 |
| 云SSD     | 100GB~32000GB | SSD 云硬盘适用于对 I/O 性能有较高要求的场景。                |
| 增强型SSD | 100GB~32000GB | 增强型 SSD 云硬盘适用于对大型数据库、NoSQL 等对时延要求很高的 I/O 密集型场景。 |
