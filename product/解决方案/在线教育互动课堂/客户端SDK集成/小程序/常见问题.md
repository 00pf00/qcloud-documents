## 常见问题

#### 1. iOS和Android在小程序中点击右上角圆圈和点击返回按钮触发事件的差异性
| 操作方式     | iOS         | Android  |
| ------------ | ----------- | -------- |
| 头部返回按钮 | 5000        | 没有事件 |
| 小程序右上角圆圈         | 5000 ，1019 | 5000     |
| 底部返回按钮 | -           | 没有事件 |

#### 2. 白板和IM能与其他端互通，音视频不能互通
确认小程序端的useCloud参数（true：云上环境 false：自研环境），检查其他端是否是同一个环境。

#### 3. 音视频房间报获取推流sig失败（CONSTANT.ROOM.ERROR_REQUEST_ROOM_SIG错误）
一般为下发的privateMapKey错误，请检查privateMapKey签名算法

#### 4. IM相关操作失败回调错误码对照表
IM [错误码对照表](https://cloud.tencent.com/document/product/269/1671)

#### 5. ```https://xxxx.xxx.xxx``` 不在以下 request 合法域名列表中
如果出现以下提示，原因是该域名没有配置在安全域名中，请参考接入文档配置。

#### 6. 音视频房间报超过最大成员数(CONSTANT.ROOM.ERROR_EXCEEDS_THE_MAX_MEMBER)

当前webrtc-room中只提供了1V1模板，所以房间内成员数最大只能是两个人，如果业务中有多人需求，则修改修改webrtc-room组件中的源码，且可能还要自定义音视频展示模板（模板承载展示音视频的位置）。

> 如果你没有小程序开发经验，请先阅读小程序开发文档。如果自定义模板，或者修改webrtc-room.js的源码，请一定要先阅读webrtc-room组件的源码。

#### 7. 自定义音视频模板

如果要自定义音视频模板，可以参考[界面定制](https://cloud.tencent.com/document/product/647/17018#CustomUI)，也可以参考[demo源码](https://github.com/TencentVideoCloudMLVBDev/MiniProgram/tree/master/wxlite/pages/components/webrtc-room)里面的1u3dtemplate, 1l3rtemplate, gridtemplate等模板。

> 如果你没有小程序开发经验，请先阅读小程序开发文档。如果自定义模板，或者修改webrtc-room.js的源码，请一定要先阅读webrtc-room组件的源码。
