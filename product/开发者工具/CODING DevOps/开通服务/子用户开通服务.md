
## 操作场景

子用户是由主账号创建的实体，有确定的身份 ID 和身份凭证，能够登陆并独立设置控制台，且具有 API 访问策略。若您希望通过腾讯云子用户使用 CODING DevOps 服务（以下简称服务），下文将按照多种场景进行指引。

1.  主账号已开通服务，需帮助子用户开通服务
2.  主账号未开通服务，需使用子用户协助主账号开通服务
3.  子用户已开通服务，但部分功能受限

## 主账号已开通服务

1. 主账号登录 [CODING DevOps 控制台](https://console.cloud.tencent.com/coding)。
2. 单击**快速创建子用户**。
![](https://help-assets.codehub.cn/enterprise/20211117145412.png)
3. 在**快速新建用户**页面填写子用户相关信息，访问方式请同时勾选**编程访问**及**腾讯云控制台访问**，密码设置可选自动生成或自定义密码，单击**创建用户**按钮后即可创建出拥有 CODING DevOps 策略的子用户。
![](https://help-assets.codehub.cn/enterprise/20211117145549.png)

>! 新建子用户时，请务必勾选访问方式的**编程访问**和**腾讯云控制台访问**，避免子用户无法登录 CODING DevOps 控制台的情况。

创建完成后，使用子用户登录[控制台](https://console.cloud.tencent.com/coding)，点击加入团队按钮。

![](https://help-assets.codehub.cn/enterprise/20211117144216.png)

按照提示填写账号信息后加入团队。

![](https://help-assets.codehub.cn/enterprise/20211117144419.png)

## 主账号未开通服务

### 使用主账号直接开通服务

请联系主账号拥有者，参考[本文](主账号开通服务.md)了解如何使用主账号直接开通服务。

### 使用子用户帮助主账号开通服务

子用户需前往[访问管理控制台](https://console.cloud.tencent.com/cam)，确认拥有 `AdministratorAccess` 权限策略，然后前往 [CODING DevOps 控制台](https://console.cloud.tencent.com/coding)。

![](https://help-assets.codehub.cn/enterprise/20211117151916.png)

点击「开通」按钮，进入主账号开通服务流程，输入主账号邮箱与验证码完成开通。

![](https://help-assets.codehub.cn/enterprise/20211117152335.png)

若主账号邮箱不存在，需进入主账号创建流程，输入密码与团队名称后开通账号。

![](https://help-assets.codehub.cn/enterprise/20211117152728.png)

若主账号邮箱已关联多个 CODING 团队，选择需要的团队即可。

![](https://help-assets.codehub.cn/enterprise/20211117153107.png)

开通成功后，子用户即可加入主账号团队。

![](https://help-assets.codehub.cn/enterprise/20211117153225.png)

## 子用户已开通服务

子用户能否继续创建新的子用户，取决于[访问管理控制台](https://console.cloud.tencent.com/cam)中所选择的策略。

![](https://help-assets.codehub.cn/enterprise/20211117153928.png)

当子用户具备 `AdministratorAccess` 策略时，控制台将出现「快速创建子用户」按钮，意味着该子用户能够创建新的子用户。

![](https://help-assets.codehub.cn/enterprise/20211117153612.png)

若子用户仅具备 `QcloudCODINGFULLAccess`，控制台中将不再出现「快速创建子用户」按钮，意味着该子用户仅具备服务的访问策略，无法继续新建子用户。

![](https://help-assets.codehub.cn/enterprise/20211117153749.png)
