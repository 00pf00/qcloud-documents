### 第一步：登录控制台
注册并通过实名认证后，登录腾讯云控制台。如果没有账号，请参考 [账号注册教程](https://cloud.tencent.com/document/product/378/17985)。

### 第二步：开通服务
单击进入 [CODING DevOps 控制台](https://console.cloud.tencent.com/coding)，进行服务授权后即可开通腾讯云 CODING 服务。

### 第三步：创建项目
进入 CODING DevOps ，单击“+”创建新项目，依次填入项目名称、地址、描述信息，其中项目地址即您的代码仓库的地址，建议勾选“启用 README.md 文件初始化项目” ，单击【新建项目】，完成创建。

### 第四步：添加成员
在腾讯云控制台进入 [访问管理-用户列表](https://console.cloud.tencent.com/cam)，单击某一子账号用户详情，单击【关联策略】，为其添加 CODING 服务。

### 第五步：配置持续部署
选择 Kubernetes 或腾讯云服务器初始化持续部署，按照指导完成云账号、应用与部署流程配置过程。

- 配置云账号：云账号是访问云资源的凭证，只有配置了相应的云账号后，CODING 持续部署才能实现对云资源的部署管理和基础设施管理。
- 配置应用：应用包含若干个应用集群、安全组和负载均衡器等，是持续部署中的基本部署单位。
- 配置部署流程：部署流程是实现持续部署最核心的模块。其强大之处在于支持阶段以任意的顺序进行组合，让部署流程具备出色的灵活性、一致性和可重复性。
- 应用与项目关联：将应用跟指定项目进行关联后，即可在对应项目中执行应用的部署流程。

### 第六步：触发持续部署
完成持续部署配置过程后，开发人员可选择手动或自动触发部署流程执行。触发完成后可以在发布单列表中查看部署状态、阶段执行详情等信息。

- 自动触发部署流程：通过设定的条件自动触发应用部署。
- 手动触发：手动提交发布单完成应用部署。
