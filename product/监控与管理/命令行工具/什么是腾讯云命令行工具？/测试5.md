## 操作场景

Kubernetes 提供了 kubectl 命令行工具用于操作集群，使用 kubeconfig 作为配置文件，默认路径为 `~/.kube/config`。如果需要使用 kubectl 管理和操作多个集群，通过 kubeconfig 配置多个集群的信息即可。

如果使用容器服务 TKE 或 EKS 集群，并且希望使用 kubectl 来管理和操作集群，只需要在集群基本信息页开启 APIServer 的外网访问或内网访问即可得到 kubeconfig (集群访问凭证)，但如果要用 kubectl 管理多个集群，通常做法是提取 kubeconfig 中各个字段的内容，然后将其合并到 kubectl 所在机器的 kubeconfig 文件中，操作繁琐且容易出错。

借助 `kubecm` 工具，可以更简单高效的将集群的访问凭证合并添加到 kubeconfig 中。本文将介绍如何利用 `kubecm` 来实现多集群的 kubeconfig 高效管理。



## 前提条件


- 已创建 TKE 或 EKS 集群，详情请参见 [创建集群](https://cloud.tencent.com/document/product/457/32189)。
- 在需要管理多集群的机器上已安装 kubectl 命令。


## 操作步骤

### 安装 kubecm

在管理多集群的机器上安装 [kubecm](https://kubecm.cloud/#/en-us/install)。


### 获取集群访问凭证

新建集群后，获取集群访问凭证步骤如下：

#### TKE 集群开启步骤

1. 登录 [容器服务控制台](https://console.cloud.tencent.com/tke2)。
2. 在左侧菜单栏中单击【集群】，进入“集群管理”页面。
3. 单击需要配置访问凭证的集群名称，进入集群信息页面。
4.

### 使用 kubecm 添加访问凭证到 kubeconfig

获取到集群访问凭证后，假设文件名为 `cls-l6whmzi3-config`，使用 `kubecm` 将其添加到 kubeconfig 中 (`-n` 可指定 context 名称):

``` bash
kubecm add -f cls-l6whmzi3-config -n cd -c
```

### 查看集群列表

通过 `kubecm` 添加了要管理和操作的集群后，通过 `kubecm ls` 可查看 kubeconfig 中的集群列表 (星号标识的是当前操作的集群):

``` bash
$ kubecm ls
```

### 切换集群

如果需要切换到其它集群操作，可以使用命令 `kubecm switch` 进行交互式切换。如下图所示：

![](https://main.qcloudimg.com/raw/3eea3d35d3a19f93906eabf60a423a0b.png)

### 移除集群

如果需要移除某个集群，可以使用命令 `kubecm delete` 进行移除。示例如下：

``` bash
$ kubecm delete bj
```
