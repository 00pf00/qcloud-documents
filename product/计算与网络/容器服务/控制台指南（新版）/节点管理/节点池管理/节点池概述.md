## 简介



为帮助您更好地管理 Kubernetes 集群内节点，腾讯云容器服务 TKE 引入节点池概念。借助节点池基本功能，您可以方便快捷地创建、管理和销毁节点，以及实现节点的动态扩缩容：
- 当集群中出现因资源不足而无法调度的实例（Pod）时，自动触发扩容，为您减少人力成本。
- 当满足节点空闲等缩容条件时，自动触发缩容，为您节约资源成本。

节点池整体架构图如下：
![](https://main.qcloudimg.com/raw/96086aafcb5d563980a489900d2060d0.png)

通常情况下，节点池内的节点均具有相同属性，包含以下几种：
- 节点操作系统。
- 节点 Kubernetes 组件启动参数。
- 节点自定义启动脚本。
- 节点 kubernetes Lable 和 taint 设置。

除此之外，TKE 将同时围绕节点池扩展以下属性：
- 节点池级别操作系统。
- 节点池级别每节点的 Pod 数上限。
- 包年包月计费类型节点池。

## 应用场景
当业务需要使用大规模集群时，推荐您使用节点池进行节点管理，以提高大规模集群易用性。下表介绍了多种大规模集群管理场景，并分别展示节点池将会在每种场景下发挥的作用：

| 场景 | 作用 |
| ---- | ---- |
| 集群存在较多异构节点（机型配置不同）| 通过节点池可规范节点分组管理 |
| 集群需要频繁扩缩容节点  | 通过节点池可降低操作成本 |
| 集群内应用程序调度规则复杂 | 通过节点池标签可快速指定业务调度规则 |
| 集群内节点日常维护 | 通过节点池可便捷操作 Kubernetes 版本升级、Docker 版本升级 |


## 功能点及注意事项

| 序号 | 功能点 | 功能说明 | 注意事项 |
| ---- | ------| -------- | ---------- |
| 1 | 创建节点池 | 新增一个节点池， 暂不支持包年包月类型。 | 单个集群不建议超过20个节点池。 |
| 2 | 删除节点池 | 删除节点池时可选择是否销毁节点池内节点。保留的节点不归属任何伸缩组和节点池。 | 删除节点池不保留节点，如需新节点可重新创建。 |
| 3 | 节点池开启弹性伸缩 | 开启弹性伸缩后，节点池内节点数量将随集群负载情况自动调整。     | 请勿在伸缩组控制台开启和关闭弹性伸缩。 |
| 4 | 节点池关闭弹性伸缩 | 关闭弹性伸缩后，节点池内节点数量不随集群负载情况自动调整。     | 请勿在伸缩组控制台开启和关闭弹性伸缩。 |
| 5 | 调整节点池大小 | 支持直接调整节点池内节点数量。若减小节点数量，将从伸缩组内随机缩容节点。 | 开启弹性伸缩后，不建议您手动调整节点池大小。|
| 6 | 调整节点池配置 | 可修改节点池名称、伸缩组节点数量范围、kubernetes label、taint。 | 修改 label 和 taint 后，会对节点池内节点全部生效。可能会造成 Pod 的重新调度，请谨慎变更。 |
| 7 | 添加已有节点 | 可添加不属于集群的实例到节点池。要求实例与集群属于同一私有网络、实例未被其他集群使用、实例与节点池配置相同（机型、计费模式），可添加集群内 node-without-pool 的节点但要求节点实例与节点池配置相同（机型、计费模式） | 无特殊情况时，不建议使用添加已有节点，推荐直接新建节点池。 |
| 8 | 移出节点池内节点   | 不支持直接移出伸缩组内节点，可移除手动添加的节点，移除后节点可选择保留到集群。 |  建议您通过调整大小形式加入或移出节点  |
| 9 | 原伸缩组转换节点池 | 支持存量伸缩组切换为节点池。转化后，节点池完全继承原伸缩组的功能，该伸缩组信息将不在展示。集群内存量所有伸缩组切换完成后，不再提供伸缩组入口。 | 操作不可逆，请熟悉节点池功能后再进行切换。 |
