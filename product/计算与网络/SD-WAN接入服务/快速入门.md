使用 SD-WAN 接入服务可以帮助您快速搭混合云网络。本文将介绍如何使用 SD-WAN 接入服务实现北京和广州两个地域的分支机构与广州地域的 VPC 网络互通。


## 背景信息

若用户 A 需实现北京（业务网段：192.168.0.0/16）和广州两个地域（10.0.0.0/16）的分支机构接入腾讯云，实现本地分支与云上网络资源互通。可以使用 Edge-100 型号的 Edge 设备，将两个分支与腾讯云网络连接，并通过云联网连接腾讯云 VPC。
![](https://main.qcloudimg.com/raw/058f189dc2d09eca32b8daeaf2cca2dc.png)

## 前提条件
- 目前 SD-WAN 接入服务处于内测阶段，如需使用，需已提交并通过 [内测申请](https://cloud.tencent.com/apply/p/v70vi3xrgr)。
- 已在广州地域创建 VPC，具体操作请参见 [快速搭建 IPv4 私有网络](https://cloud.tencent.com/document/product/215/30716)。
- 已创建云联网实例，并与广州地域的 VPC 关联，具体操作请参见 [新建云联网实例](https://cloud.tencent.com/document/product/877/18752)。


## 操作步骤

使用 SD-WAN 接入服务实现北京和广州两个地域的分支机构与广州地域的 VPC 网络互通的操作流程如下：
![](https://main.qcloudimg.com/raw/37d085de44ededb993012263d0e712a2.png)

1. [申请 Edge 设备](#1)：分别为北京和广州分支申请 Edge 设备。
2. [安装 Edge 设备](#2)：Edge 设备支持即插即用。
3. [配置网络连接](#3)：分别为北京和广州分支的 Edge 设备配置网络连接。
4. [关联云联网](#4)：将北京和广州地域的 SD-WAN 接入服务实例、腾讯云 VPC 关联至云联网，实现网络互通。
5. [配置防火墙](#5)：分别为北京和广州地域的  SD-WAN 接入服务实例创建防火墙。



<span id = "1"></span>

### 步骤1：申请 Edge 设备

1. 登录 [SD-WAN 接入服务控制台](https://console.cloud.tencent.com/sas/edge)。
2. 在 “Edge 设备”页面上方，单击【新建】。
3. 在“新建 Edge” 对话框中，编辑 Edge 名称、并选择带宽峰值、带宽时长、购买数量等购买信息，并填写收货信息，单击【确定】。
![](https://main.qcloudimg.com/raw/1813ea942315e8134c84e07e97e7765e.png) 
4. 按照2 - 3步骤分别为北京和广州两个地域的分支购买 Edge 设备。
 >?创建订单后，控制台将自动为您创建 SD-WAN 接入服务实例。

<span id = "2"></span>
### 步骤2：安装 Edge 设备
请按以下步骤，分别为北京和广州地域分支机构安装 Edge 设备：
1. 收到 Edge 设备后，请检查配件是否完整。配件详情请参见 [硬件特性说明](https://cloud.tencent.com/document/product/1277/47251)。
2.  将 WAN 口 和 Modem 相连，LAN 口和本地客户端相连。
   >?Edge 设备支持插电即用，若您要通过本地 Web 控制台（即192.168.3.1）配置 Edge 设备，请参见 [在本地 Web 控制台配置 Edge 设备](https://cloud.tencent.com/document/product/1277/47252)。

<span id = "3"></span>
### 步骤3：配置网络连接
请按以下步骤，分别为北京和广州分支的 Edge 设备配置网络连接：
1. 配置 WAN 接口。
   - 方式1：通过本地 Web 控制台（即192.168.3.1）配置。当 Edge 设备的 4G 网络正常使用（即插电后，LTE 灯长亮），且猫正常使用时，可以使用本地 Web 控制台配置 WAN 口。请参见 [在本地 Web 控制台配置 Edge 设备](https://cloud.tencent.com/document/product/1277/47252)。
   - 方式2：在 SD-WAN 接入服务控制台配置：
     1. 登录 [SD-WAN 接入服务控制台](https://console.cloud.tencent.com/sas/edge)，并在实例列表中单击 [步骤1](#1) 中创建的 SD-WAN 接入服务 ID。
     2. 在左侧导航栏，选择【设备配置】>【接口配置】。
     3. 在“接口配置”页面查看物理接口基本信息，系统自动为 WAN1 接口和 WAN2 接口进行以下配置。
     >?若默认设置不能满足您的业务需求，可修改 WAN 接口设置，详情请参见 [配置 WAN 接口](https://cloud.tencent.com/document/product/1277/47271)，本示例中使用默认配置。
     >
![](https://main.qcloudimg.com/raw/36e1164bd6d691c70dd08254669e1636.png) 
2. 配置 LAN 接口。
   1. 登录 [SD-WAN 接入服务控制台](https://console.cloud.tencent.com/sas/edge)，并单击目标实例 ID。
   2. 在左侧导航栏选择【设备配置】>【接口配置】。
   3. 在“物理接口”页面，单击【LAN1】。
   4. 查看 LAN 接口基本信息。系统自动为 LAN 1、LAN 2 和 LAN3 接口分别配置了一个逻辑接口。
   5. 若默认设置不能满足您的业务需求，可修改 LAN 接口设置请参见 [配置 LAN 接口](https://cloud.tencent.com/document/product/1277/47270)，本示例中使用默认配置。
![](https://main.qcloudimg.com/raw/a53a302cab76b543f81afdd9493e426c.png) 
3. 配置路由表。
   1. 在左侧导航栏，单击【路由表】。
   2. 在“路由表”页面，单击【新建】。
   2. 在“新建静态路由”页面配置以下信息，单击【确定】。

### 步骤4：关联云联网
请按以下步骤，分别将北京和广州地域的 Edge 设备所对应的 SD-WAN 接入服务实例，关联至已关联腾讯云 VPC 的云联网。

1. 登录 [SD-WAN 接入服务控制台](https://console.cloud.tencent.com/vpc/edge)。
2. 在 “Edge 设备”页面，勾选目标 Edge 设备实例，并在页面上方单击【关联云联网】。
   >?仅设备状态为“运行中”的 Edge 设备才能关联云联网。
3. 在“关联云联网”对话框中，选择云联网实例，并单击【确定】。
4. 检查云联网路由表。
   查看分支的路由与 VPC 子网路由策略是否有效，若所关联的网络实例网段有冲突，则会产生失效路由。
   1. 在“Edge 设备”页面，找到目标 Edge 设备实例的“关联云联网”列，单击云联网 ID。
   2. 在云联网详情页面，单击【路由表】页签。
   3. 在“路由表”页签中，检查是否存在状态为“失效”的路由策略。若存在，则根据 [路由冲突原则](https://cloud.tencent.com/document/product/877/18679#.E8.B7.AF.E7.94.B1.E9.99.90.E5.88.B6) 修改路由表并启用路由，详情请参见 [启用路由](https://cloud.tencent.com/document/product/877/18750)。



<span id = "5"></span>
### 步骤5：配置防火墙
创建 SD-WAN 接入服务实例后，系统将自动为您创建默认防火墙。默认防火墙规则为放通全部出接口，禁止全部入接口的防火墙。本示例中使用默认防火墙，若默认防火墙不能满足您的需求，请重新创建防火墙（具体操作请参见 [创建防火墙](https://cloud.tencent.com/document/product/1277/47266)）后，再按以下步骤更换防火墙。
1. 登录 [SD-WAN 接入服务控制台](https://console.cloud.tencent.com/sas/edge)，并单击目标实例 ID。
2. 在“基本信息”页面的“防火墙”字段右侧，单击【更换】。
 ![](https://main.qcloudimg.com/raw/d62b1dccacab3bbadf33a989b60cdb83.png)
3. 选择已创建的防火墙，单击【确认】。


### 结果验证
使用北京和广州分支机构的客户端访问腾讯云 VPC 中部署的云资源，验证配置是否生效。
