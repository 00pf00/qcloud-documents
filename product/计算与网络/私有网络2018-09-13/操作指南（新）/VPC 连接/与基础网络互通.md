基础网络是腾讯云上所有用户的公共网络资源池。所有云服务器的内网 IP 地址都由腾讯云统一分配，无法自定义网段划分、IP 地址。私有网络是用户在腾讯云上建立的一块逻辑隔离的网络空间。在私有网络内，用户可以自由定义网段划分、IP 地址和路由策略。与基础网络相比，私有网络更适合有网络自定义配置需求的场景。当业务同时存在于基础网络和私有网络时，可能涉及如下场景：
+ **基础网络和私有网络业务互通**，那么您可以选择如下方式实现：
    + 基础网路互通：实现同地域同账号下基础网络 CVM 与 VPC 内云资源（云服务器、负载均衡、云数据库等）的互通，且 VPC 内的云服务器只能访问关联该 VPC 的基础网络云服务器，不能访问基础网络其他云资源。具体请参见 [基础网络互通]()。
    + 终端连接：实现同地域同账号下 VPC 内 CVM 单向访问基础网络内非云服务器资源，终端连接支持的基础网络云资源包括：CLB、MySQL、Memcached、Redis、MariaDB、SQL Server、PostgreSQL、MongoDB、TDSQL。
    >?
    >+ 终端连接的原理是：将基础网络实例 IP 与私有网络内 IP 建立映射，访问该私有网络 IP 即访问基础网络实例。
    >+ 终端连接不支持跨地域、跨账号，如您有建立终端连接的需要，请提交 [工单申请](https://console.cloud.tencent.com/workorder/category)。
+ **基础网络业务迁移至私有网络**，目前腾讯云提供单实例迁移，以及混访模式下的业务平滑迁移：
    + 单实例迁移目前支持：CVM、MySQL、MariaDB、TDSQL、Redis、MongoDB 等。
    + 混访模式的平滑迁移：混访是指云资源在进行网络迁移时，同时保持基础网络和私有网络的IP地址，可同时被基础网络和私有网络中的 CVM 访问，确保业务不中断，例如云数据库 MySQL、MariaDB、TDSQL、Redis、MongoDB、COS 等支持混访。
    >?更多详细内容，请参见 [基础网络迁移到私有网络]()。
