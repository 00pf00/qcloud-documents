
通过控制台或腾讯云命令行均可以完成函数创建。


## 通过控制台创建函数

1. 登录 [云函数控制台](https://console.cloud.tencent.com/scf)，在左侧选择函数服务。
2. 在主界面上方选择期望创建函数的地域。单击【新建】按钮，进入函数创建流程。
3. 可选择使用【空白函数】或选择使用【函数模板】来新建函数。
 - 使用【空白函数】时，通过填写必填的函数名称、运行环境来完成函数的创建。
 - 使用【函数模板】时，通过填写必选的函数名称即可创建函数，运行环境等配置依赖函数模板中的配置来创建。



## 更多创建方式
您可根据实际需求，选择更多方式创建函数：
- 使用命令行工具，可参考 [基于 CLI 开发 Hello World Web 服务](https://cloud.tencent.com/document/product/583/37510) 教程使用 SCF CLI 创建函数/调用/部署。
- 使用 VS Code ，可参考 [VS Code插件](https://cloud.tencent.com/document/product/583/36208) 教程使用 VS Code 插件创建函数/调试/部署。



## 更新函数配置

### 通过控制台更新函数配置

1. 登录 [云函数控制台](https://console.cloud.tencent.com/scf)，在左侧选择函数服务。
2. 在主界面上方选择期望更新的函数所在地域。在列表页中单击期望更新的函数，进入函数详情页面。
3. 切换至函数配置子页面，并单击右上角的【编辑】按钮，进入编辑模式。
4. 可根据需求修改函数的描述、配置内存、超时时间、环境变量和网络配置等信息。
5. 修改完成后，单击【保存】按钮保存修改后的配置。若想取消，通过单击【取消】按钮，取消修改的配置。

### 通过 SCF 命令行更新函数配置

如需修改函数配置，可以直接修改 template.yaml ，然后执行 `scf deploy` ，部署函数详情[参考教程](<https://cloud.tencent.com/document/product/583/33451>)。

如果修改了 template.yaml 的函数名，部署时会新增该函数；

注意：函数运行时无法修改

```yaml
test:
      Type: TencentCloud::Serverless::Function
      Properties:
        CodeUri: ./
        Type: Event
        Description: This is a template function
        Environment:
          Variables:
            ENV_FIRST: env1
            ENV_SECOND: env2
        Handler: index.main_handler
        MemorySize: 128
        Runtime: Nodejs6.10
        Timeout: 3
```



## 更新函数代码

### 通过控制台更新函数代码

1. 通过 [云函数控制台](https://console.cloud.tencent.com/scf)，在左侧选择函数服务。
2. 在主界面上方选择期望更新的函数所在地域。在列表页中单击期望更新的函数，进入函数详情页面。
3. 切换至函数代码子页面，针对脚本类语言，可以看到函数代码编辑器，针对非脚本类语言，仅有函数上传方式。
4. 可通过代码编辑器直接编辑入口函数，或者切换代码上传方式，选择通过 zip 包上传，或通过对象存储 COS 上传的方式提交函数。
5. 修改完成后，单击【保存】按钮保存或提交新的代码。若想取消，通过单击【取消】按钮取消修改。

### 通过 SCF 命令行更新函数代码

在本地修改函数代码后，执行 `scf deploy` 即可完成代码更新，部署函数详情[参考教程](<https://cloud.tencent.com/document/product/583/33451>)。



## 复制函数

### 操作场景

通过控制台可以完成函数复制，您可以通过跨地域和跨命名空间完成函数复制，复制时可以仅复制函数代码，也可以同时复制函数代码及函数配置。

### 功能介绍

通过复制函数功能，我们可以很方便的在同一地域创建出名字不同的代码相同的函数，也可以在不同地域创建相同名字的函数。对于代码重复程度较高的函数，我们可以通过复制功能快速创建函数，修改代码，快速的实现多个有细微差异的云函数。

我们将被复制的云函数成为源函数，复制到的函数称为目标云函数。

#### 复制源及复制目标

您可以通过选择任意地域任意命名空间的云函数作为源函数发起复制操作。源函数的 $LATEST 版本将作为复制内容被复制，源函数的非 $LATEST 版本暂时不支持复制。

复制的目标函数可以选择任意地域、任意命名空间及自定义命名。在选择的地域、命名空间内若有同名函数，复制操作将覆盖同名函数，且复制的目标函数仅生成或更新 $LATEST 版本。

####复制内容

云函数可复制的内容包括代码及配置。
- 函数代码：包括函数的代码包、运行环境、执行方法。
- 函数配置：函数的内存、超时、描述、环境变量、网络、日志等配置内容。**函数配置不包括触发器配置**。

函数复制时，可选择以下两种复制方式：
- 仅复制代码：仅将源函数的代码复制到目标函数的代码，目标函数如果存在则使用原配置，否则使用默认配置。目标函数如果存在，运行环境需要与源函数运行环境相同。
- 复制代码及配置：将源函数的代码与配置均复制到目标函数。目标函数如果存在，运行环境需要与源函数运行环境相同。

>! 
> - 如果源函数与目标函数处于不同地域，在复制代码及配置时，函数配置中的网络、日志配置将无法复制到目标函数中。
> - 由于在跨地域时不具有相同的对象，若有地域属性的配置项，将会导致配置无法复制至云函数。如需复制至云函数，您可以在完成复制后，手动编辑云函数，修改所需配置。

### 操作步骤

1. 登录 [无服务器云函数控制台](https://console.cloud.tencent.com/scf)。
2. 在左侧导航栏中，选择【[函数服务](https://console.cloud.tencent.com/scf/list)】，进入函数服务管理页面。
3. 在“函数服务”上方，选择期望更新的函数所在地域，查看该地域下的所有函数。
4. 在函数列表中，选择需要复制的源函数行，单击“操作”栏中的【复制】。
5. 在弹出的“函数复制”窗口中，选择所需的目的函数的地域、命名空间并输入目标函数名，可选填目标函数描述，并选择【仅复制函数代码】或【复制函数代码或配置】。
6. 单击【确认】完成复制。如果目标函数已存在，请在警告窗口中再次确认，或取消后重新修改函数名。
