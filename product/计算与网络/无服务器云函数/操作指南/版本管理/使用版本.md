在完成云函数的版本发布后，我们可以通过调用指定版本的云函数使用版本。

## 查看版本

1. 登录  [无服务器云函数控制台](https://console.cloud.tencent.com/scf)。
2. 在左侧导航栏中，选择 “函数服务”，进入服务信息页面。
3. 选择待查看云函数版本的函数名，进入函数信息页面。如下图所示：
![](https://main.qcloudimg.com/raw/62565ae1fbff715471796a61ec7fe337.png)
4. 在函数信息页面的右上方，选择需要查看的版本，即可切换到对应版本，查看该版本的云函数配置和代码。
>? 切换到版本后，函数配置、代码、触发器、日志及监控的显示内容将显示为对应版本的内容。其中，函数配置及代码保持发布时状态，无法修改；触发器可以在不同版本上进行不同的配置；日志和监控分别显示对应版本的具体调用日志和监控数据。

## 使用版本

版本功能主要用于对于函数配置和代码的固化，避免开发调试及测试时，对业务的影响。建议在完成开发和测试后，发布版本，并在已发布的版本上绑定触发器，运行实际业务。
>? $LATEST 版本为开发和测试使用的版本，用于代码的进一步开发和调试。

### 版本的触发器

目前云函数已发布的版本均可以独立绑定触发器。同一函数，版本与版本之间独立，每个触发器都独立触发函数运行。
针对触发器数量，有以下限制：
- 云函数触发器页面配置的触发器仅可配置 2 个。
例如，您已在云函数触发器中创建一个 demo1 函数，并在该函数的版本1中绑定两个 COS 触发器，在其他版本中则不能增加 COS 触发器。如需增加触发器，只能增加定时或 CMQ Topic 触发器。
- 其他产品或服务中配置的云函数触发器不受此限制。
例如，API 网关触发云函数，可配置任意多个 API 触发同一个云函数。

>? 如需增加触发器的配额数量（即配额提升），可通过 [提交工单](https://console.cloud.tencent.com/workorder/category) 申请。

### 云 API 触发版本

使用云 API InvokeFunction 接口触发云函数调用时，可通过可选参数 Qualifier 指定需要触发的具体版本。如果没有此参数，默认触发 $LATEST 版本。
