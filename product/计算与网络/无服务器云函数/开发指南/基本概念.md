用户在使用 SCF 平台支持的语言编写代码时，需要采用一个通用的范式，包含以下核心概念：

## 执行方法
执行方法决定了 SCF 平台从何处开始执行您的代码，您需以**文件名.执行方法名**的形式进行设置。SCF 平台在调用云函数时，会通过寻找执行方法来开始执行您的代码。
例如，用户设置的执行方法为 `index.handler`，则 SCF 平台会首先寻找代码程序包中的 `index` 文件，并找到该文件中的 `handler` 方法开始执行。

用户在编写执行方法时需遵循平台特定的编程模型，如下所示：
```
def method_name(event,context): 
    ...
    return some_value
```
该模型中指定固定的入参：事件数据 event 和环境数据 context。执行方法应对参数进行处理，并且可任意调用代码中的任何其他方法。


## 函数入参
函数入参，是指函数在被触发调用时所传递给函数的内容。通常情况下，函数入参包括 event 入参和 context 入参两部分。但根据开发语言和环境的不同，入参个数可能有所不同。不同开发语言在入参上的具体差异，详情请参见 [开发语言说明](https://cloud.tencent.com/document/product/583/11061)。

## 函数返回
SCF 平台会获取到云函数执行完成后的返回值，并根据下表中不同的触发方式进行处理。

	<tr>

	<th>描述</th>

		<li>通过同步方式触发的云函数，在函数执行期间，SCF 平台不会返回请求。在函数执行完成后，会将函数返回值封装为 JSON 格式并返回给调用方。</li>
		</ul>
	</td>

	<tr>
	<td>异步触发</td>
	<td>
		<ul class="params">

			<li>在函数执行完成后，函数的返回值会封装为 JSON 格式后存储在日志中。</li>
			<li>如果用户需要在异步触发后获取函数的返回值，可在函数执行完成后，通过记录请求返回中的 requestId 查询日志，即可获取到此次执行的函数返回值。</li>
		</ul>
	</td>

为保证针对各开发语言和环境的统一性，函数返回会使用 **JSON 数据格式统一封装**。当函数中返回具体值时，通常返回特定的数据结构。例如 ：
- Python 环境：通常为简单数据结构或 `dict` 数据结构。
- Node.js 环境：通常为 `JSON Object`。
- PHP 环境：为 `Array` 结构。
- GO 环境：为简单数据结构或带有 JSON 描述的 `struct`。

SCF 平台在获取到例如以上运行环境函数的返回具体值后，将会对返回的数据结构进行 JSON  化，最终返回 JSON 内容到调用方。需注意以下几点：
- 需要确保函数的返回值均可以 JSON 化，若返回的对象不具备 JSON 化方法，将会导致 SCF 平台在 JSON 化操作时失败并报错。
- 返回值也无需在 return 前自行 JSON 化，否则会导致输出的字符串被再次 JSON 化操作。

## 异常处理
若函数在调试和运行过程中出现异常，SCF 平台会尽最大可能捕获异常并将异常信息写入日志中。函数运行产生的异常包括捕获的异常（Handled error）和未捕获的异常（Unhandled Error）。

### 处理方式
本文提供以下三种抛出异常方式，您可根据实际需求选择在代码中进行异常处理。

> 1. 新建函数并复制以下函数代码，不添加任何触发器。
> 2. 单击控制台【测试】按钮，选择 “Hello World” 测试示例进行测试。

### 返回错误信息
当用户的代码逻辑中未包含异常及错误处理时，SCF 平台会在用户使用时尽可能的捕获错误。当出现平台也无法捕获错误的情况，例如用户函数在运行过程中突然退出时，SCF 平台将会返回一个通用的错误信息。您可根据下表中的常见错误信息分析原因：

| 错误场景            | 返回消息                                                     |

| 依赖模块并不存在    | {global name 'xxx' is not defined}                           |
| 超时                | {"time out”}                                                 |

SCF 平台会将函数调用的所有记录及函数代码中的输出全部存储在日志中，请使用编程语言中的打印输出语句或日志语句生成输出日志，方便调试及故障排除时使用。

### 获取日志
您可以通过以下途径获取函数运行日志：
- **SCL CLI 命令行工具**：执行 `scf logs` 命令，详情请参见 [日志查看](<https://cloud.tencent.com/document/product/583/36352>)。
- **SCF 控制台**：在函数代码页面，单击【测试】按钮同步调用函数，执行完成后会直接在控制台展示本次调用的日志。
-  **触发器调用**：该函数的日志选项卡中会展示函数每一次被调用产生的日志。
- **SCF API**：通过 GetFunctionLogs 接口获取函数日志。
-  **Invoke API 同步调用**：可在返回值的 logMsg 字段中获取本次调用的日志。

## 注意事项
由于云函数的特点，您须以**无状态**的风格编写您的函数代码。本地文件存储等函数生命周期内的状态特征在函数调用结束后将随之销毁。因此，持久状态建议存储在关系型数据库 TencentDB、对象存储 COS、云数据库 Memcached 或其他云存储服务中。
