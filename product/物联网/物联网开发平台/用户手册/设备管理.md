
在物联网开发平台的某个项目中成功创建设备后，您可以在控制台管理和查看具体设备信息。


## 管理项目下的设备
### 操作步骤
1.	登录 [物联网开发平台](https://console.cloud.tencent.com/iotexplorer)。
2.	在实例概览页面中，找到需要打开的公共实例或企业实例，单击【进入实例】即可跳转到实例详情页面。
 ![](https://main.qcloudimg.com/raw/d0b5788338d2b15bb08af4287a9d1bf4.png)
3.	在项目管理页面中，找到需要打开的项目，单击项目进入项目详情页面。
 ![](https://main.qcloudimg.com/raw/57daee3e5bba0f80325e8dda3176f398.png)
4.	在左侧导航栏，选择【设备量产】>【设备管理】，进入设备管理页。
 ![](https://main.qcloudimg.com/raw/9ee46a8e609314fe5a9e467ff5f9f9ea.png)

### 操作说明

| **任务**                 | **操作步骤**                                                 |
| ------------------------ | ------------------------------------------------------------ |
| 查看某个产品下的设备信息 | 在页面左上方下拉菜单中选择某个产品。<br>您可查看设备当前状态：<br>未激活：设备未接入物联网开发平台。您可下载设备 SDK 进行设备开发，将设备接入物联网平台进行激活。<br>在线：设备已激活，与物联网开发平台成功连接。<br>离线：设备已激活，与物联网开发平台断开连接。 |
| 搜索设备                 | 输入设备名称搜索具体设备，支持模糊搜索。                     |
| 查看设备详情             | 在列表中找到对应设备，单击【查看详情】进入设备详情页。         |
| 删除某个设备             | 在列表中找到对应设备，单击【删除】即可进行删除。删除设备后，该设备证书信息将会失效，设备在物联网平台上的数据记录也会被删除。 |



## 查看具体设备信息
在设备列表中，单击设备对应的查看按钮，即可进入设备详情页。
![](https://main.qcloudimg.com/raw/51274bdddb17cebfaf450629268235f0.png)

### 操作说明

| **任务**           | **操作说明**                                                 |
| ------------------ | ------------------------------------------------------------ |
| 查看设备信息       | 查看设备基本信息：<br>- 设备密钥：使用密钥认证需要将此信息烧录到设备端。<br> - 产品 ID：唯一标识，需要烧录到设备端。<br>- 激活时间：设备第一次连接开发平台的时间。<br> - 最后上线时间：设备最后一次连接开发平台的时间。<br>- 设备状态：如果设备在线，则显示为“在线”，如果设备离线，则显示为“离线”，如果设备从未连接开发平台，则显示为“未激活”等内容。 |
| 查看设备属性       | 查看该设备的数据模板功能项：<br>- 变量标识符：对应该设备的数据模板中的标识符。<br> - 变量名称：对应数据模板中的“功能名称”。<br>- 历史数据：单击【查看】，即可查询该功能项的历史上报数据。<br>- 变量类型：对应数据模板中的“数据类型”。<br>- 最新值：当设备在向云端上报数据时，只要某个功能的最新上报值发生变化，最新值列都会立刻显示设备上报的最新值。<br>- 更新时间：指最新值的变化时间，一般是设备上报该功能的发生时间。查看某个功能的历史上报数据。按时间展示该功能上报到云端的历史数据，验证上报的数据是否正确。 |
| 查看设备日志       |查看该设备上行到云端，并从云端接收的信息。<br>- 上行：上行表示设备端向云端上报的数据。<br>- 下行：下行表示云端向设备端发送的数据。 |
| 查看设备事件       | 查看该设备上报到云端的事件信息。<br>- 事件的定义：在数据模板中定义管理。<br>- 事件类型：系统将事件类型分为三种，分别是告警、故障、信息。 |
| 查看设备行为       | 查看该设备的行为信息。<br>- 行为的定义：在数据模板中定义管理。<br>- 行为描述：用于描述复杂的业务逻辑，可添加多个调用参数和返回参数，可用于让设备执行某项特定的任务。例如：开锁动作需要知道具体用户在何时开锁，以及锁的状态情况等。 |
| 查看设备上下线日志 | 查看该设备连接云端与断开连接的日志记录。 |
