## 操作场景
符号表是内存地址与函数名、文件名、行号的映射表。符号表元素为`<起始地址> <结束地址> <函数> [<文件名:行号>]`。
为了能快速并准确地定位用户 APP 发生卡顿、内存对象分配等代码位置，QAPM 使用符号表对程序堆栈进行解析和还原。本文将指导您如何使用配置上传符号表。


## 前提条件
已下载 [ 符号表脚本](http://qapm-1253358381.cosgz.myqcloud.com/QAPM_SDK_Outer_v3.0.3.zip)。


## 操作步骤
QAPM 提供以下两种方式上传符号表。  

### 自动配置：XCode + sh 脚本
1. 在 Xcode 工程对应 Target 的 Build Phases 中新增 Run Scrpit Phase。
![](https://main.qcloudimg.com/raw/458f0327ec86973b5803c399b2ca45c1.png)
![](https://main.qcloudimg.com/raw/a50693dca612b64594bf2ba3c42227ee.png)
2. 打开 QAPM_Upload_Dsym.sh，复制所有内容，在新增的 Run Scrpit Phase 中粘贴。
3. 修改新增的 Run Scrpit 中的 "your appkey" 为您的 App Key。
![](https://main.qcloudimg.com/raw/918a8d585e6b6015bd031c977d85238b.png)
4. 脚本默认在 Debug 模式及模拟器编译情况下不会上传符号表，在需要上传的时候，请修改下列选项：
 -  Debug模式编译是否上传，1＝上传 0＝不上传，默认不上传
    UPLOAD_DEBUG_SYMBOLS=0
 - 模拟器编译是否上传，1＝上传 0＝不上传，默认不上传
    UPLOAD_SIMULATOR_SYMBOLS=0
5. 至此，自动上传符号表脚本配置完毕，QAPM 会在每次 Xcode 工程编译后自动完成符号表配置工作。


### 手工上传
在相应的个例数据页面上的上传符号表入口进行上传符号表。
