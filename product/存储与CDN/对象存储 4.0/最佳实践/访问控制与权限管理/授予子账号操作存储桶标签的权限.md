## 简介

假设企业帐号 CompanyExample（OwnerUin 为100000000001，Owner_appid 为1250000000）下有一个子账号 Developer，企业帐号 CompanyExample 需要授予该子账号拥有标签的操作权限。下面将为您详细介绍如何进行授权操作。

## 操作步骤


### 登录控制台
1. 使用企业账号 CompanyExample 登录到 [访问管理控制台](https://console.cloud.tencent.com/cam/policy)，进入到策略配置页面。
2. 授予子账号 Developer 拥有标签的操作权限：
 - [授予子账号标签全读写权限](#step1)
 - [授予子账号标签只读权限](#step2)
 - [授予子账号标签只写权限](#step3)
 - [授予子账号标签特定操作权限](#step4)


<span id="step1"></span>
### 授予子账号标签全读写权限

1. 在 [访问管理](https://console.cloud.tencent.com/cam/policy) 策略配置页面右侧的搜索框中，输入“标签”，搜索出标签相关策略。
![](https://main.qcloudimg.com/raw/5c061273480cafff17725bb9e45953ac.png)
2. 找到 QcloudTAGFullAccess，在其右侧操作一栏下，单击【关联用户/组】，进行关联用户 Developer 操作。
3. 勾选 Developer 用户，并单击【确定】，即可授予子账号 Developer 拥有标签的全读写权限。
![](https://main.qcloudimg.com/raw/2d5aa7ba1fcba97466d2f21f40319772.png)


<span id="step2"></span>
### 授予子账号标签只读权限

1. 在 [访问管理](https://console.cloud.tencent.com/cam/policy) 策略配置页面右侧的搜索框中，输入“标签”，搜索出标签相关策略。
![](https://main.qcloudimg.com/raw/63bd6bc4a2f05567410200db31ae828c.png)
2. 找到 QcloudTAGReadOnlyAccess，在其右侧操作一栏下，单击【关联用户/组】，进行关联用户 Developer 操作。
3. 勾选 Developer 用户，并单击【确定】，即可授予子账号 Developer 拥有标签的只读权限。
![](https://main.qcloudimg.com/raw/2d5aa7ba1fcba97466d2f21f40319772.png)


<span id="step3"></span>
### 授予子账号标签只写权限

由于目前访问管理预设策略中只支持**标签全读写**和**标签只读权限策略**，如果需要授予子账户**只写标签权限策略**时，可以通过新建自定义策略的形式创建。企业账号 CompanyExample 可以新建自定义策略，创建标签只写权限策略，并关联至相应的子账号下。
详细操作步骤如下：

1. 进入 [访问管理](https://console.cloud.tencent.com/cam/policy) 策略配置页面。
2. 单击【新建自定义策略】>【按照策略语法创建】。
3. 您可以选择空白模板或者已有的标签模板创建。
![](https://main.qcloudimg.com/raw/a217975ec777820eb21480db22b19fca.png)
4. 单击【下一步】，输入策略名称，并在模板编辑框中将策略语法修改为如下内容。
```
{
    "version": "2.0",
    "statement": [
        {
            "action": [
                "tag:Put*"
            ],
            "resource": "*",
            "effect": "allow"
        }
    ]
}
```
5. 单击【创建策略】，创建标签只写权限策略。
6. 将新建的策略关联子账号。

<span id="step4"></span>
### 授予子账号标签特定操作权限

如果需要授予标签特定操作权限，同样需要通过新建自定义策略实现。企业账号 CompanyExample 可以通过在 [新建自定义策略](https://console.cloud.tencent.com/cam/policy/createV3) 页面中创建新的权限策略，并关联至相应的子账号。这里可以根据业务需要，通过策略生成器或者策略语法创建。

#### 通过策略生成器创建
1. 进入 [访问管理](https://console.cloud.tencent.com/cam/policy) 策略配置页面。
2. 单击【新建自定义策略】>【按策略生成器创建】。
3. 您可以根据业务需要勾选相应的操作授予子账户相应的权限。如下图展示了对子账户授予创建标签 CreateTag 的权限。
![](https://main.qcloudimg.com/raw/7222649240c8dcd28a0741e3bc5032cd.png)
4. 单击【添加声明】，配置的策略将展示在页面下方。
![](https://main.qcloudimg.com/raw/781cc4bc71dfb8820b28a91538afb5a3.png)
5. 单击【下一步】，输入策略名称。
5. 单击【创建策略】，即可创建该策略。
6. 将新建的策略关联子账号。

#### 通过策略语法创建
1. 进入 [访问管理](https://console.cloud.tencent.com/cam/policy) 策略配置页面。
2. 单击【新建自定义策略】>【按照策略语法创建】。
3. 您可以选择空白模板或者已有的标签模板创建。
![](https://main.qcloudimg.com/raw/a217975ec777820eb21480db22b19fca.png)
4. 单击【下一步】，输入策略名称，并对模板中的策略语法进行修改，将`action`修改为指定的标签`action`名称。标签`action`名称可以前往访问管理的 [API 概览文档](https://cloud.tencent.com/document/product/651/18362) 中获取。
以下示例展示了对子账户授予创建标签`CreateTag`的权限。
```
{
    "version": "2.0",
    "statement": [
        {
            "effect": "allow",
            "action": [
                "name/tag:CreateTag"
            ],
            "resource": [
                "*"
            ]
        }
    ]
}
```
5. 完成上述步骤之后，单击【创建策略】，即可创建该策略。
6. 将新建的策略关联子账号。
