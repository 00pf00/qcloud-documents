下面基于 [搭建临时密钥服务](https://cloud.tencent.com/document/product/436/19134) 一文中提到的通过设置 STS SDK 的 Policy 参数，来实现不同的权限控制。


## 常用的 Policy

### 完全权限

拥有该账号下 COS 资源的所有权限，**不推荐使用**。

```
 {
    "version": "2.0",
    "statement":
     {
         "effect": "allow",
         "action": "cos:*",
         "resource": "*"
     }
}
```

### 拥有所有 COS 资源的读权限

```
{
    "version": "2.0",
    "statement":
     {
         "effect": "allow",
         "action":  [
                    "cos:List*",
                    "cos:Get*",
                    "cos:Head*",
                    "cos:OptionsObject"
                ],
         "resource": "*"
     }
}
```

### 指定前缀的文件的读写权限：

假设您的 appid 为 8000001，资源地域为 ap-shanghai，Bucket 为 Bucket1，文件前缀为 /dir1/test*

```
{
    "version": "2.0",
    "statement":
     {
         "effect": "allow",
         "action": "cos:*",
         "resource": "qcs::cos:ap-shanghai:uid/8000001:prefix//8000001/Bucket1/dir1/test*"
     }
}
```

更多 Policy 设置方式请参考 [权限设置相关案例](https://cloud.tencent.com/document/product/436/12514) 以及 [CAM 服务](https://cloud.tencent.com/document/product/598/10583)。
